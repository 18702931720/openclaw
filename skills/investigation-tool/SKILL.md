# 🔍 情报调查工具 (Investigation Tool)

## 一、技能定位

纯 AI 驱动的结构化背调框架，不依赖任何外部代码执行或专用 API，完全基于 OpenClaw 内置工具（searxng、web_fetch）完成。

**适用场景：**
- 供应商/客户背调
- 竞争对手分析
- 合作伙伴尽职调查
- 人物背景核查
- 事件脉络梳理

**触发词：** 调查、查背景、做尽调、背调、OSINT、情报分析

---

| ⑤ | 核心人物 | [对象] CEO 创始人 高管 团队 董事 | 标准+ |

## 三、置信度标注体系

每条信息标注可信度等级：

| 等级 | 标注 | 来源 |
|------|------|------|
| 0.9 | ✅ 确认 | 官方渠道（官网、公告、政府公示） |
| 0.7 | 🟡 可靠 | 权威媒体（财新、21世纪经济、证券时报等） |
| 0.5 | 🟠 一般 | 一般媒体/自媒体 |
| 0.3 | 🔴 待验证 | 论坛/社交媒体的未证实信息 |

## 四、工作流程

### Phase 1 — 初始化

确定调查对象和深度：

| 深度 | 角度数 | 搜索角度 |
|------|--------|---------|
| 基础版 | 3个 | 基本信息 + 司法风险 + 负面信息 |
| 标准版 | 5个 | 基本信息 + 财务 + 股权 + 人物 + 风险 |
| 深度版 | 7个+ | 全角度 + 交叉验证 |

### Phase 2 — 多源搜索

按选定角度依次搜索，每个角度搜索后立即抓取前 2-3 个高价值链接全文。

**实战技巧：**
- 上市公司优先：新浪财经、东方财富、同花顺、巨潮资讯
- 非上市公司：行业网站、头条搜索、微信公众号、政府公示
- 天眼查/企查查被拦截：换百度爱企查、国家企业信用信息公示系统

### Phase 3 — 实体提取

从搜索结果中提取 6 类实体：

| 实体类型 | 示例 |
|---------|------|
| 人物 | 创始人、高管、股东、关键联系人 |
| 公司 | 主体公司、子公司、关联公司、竞争对手 |
| 地点 | 总部、分支机构、主要市场 |
| 事件 | 融资、并购、诉讼、处罚、合作 |
| 资金 | 融资金额、估值、营收、罚款 |
| 关系 | 投资、控股、合作、诉讼、竞争 |

### Phase 4 — 关系图谱构建

整理为关系列表：
```
[主体] ──(关系类型)──> [目标] [置信度]

示例：
A公司 ──(控股60%)──> B公司 0.9
张三 ──(创始人/CEO)──> A公司 0.9
A公司 ──(诉讼被告)──> C公司 0.7
```

### Phase 5 — 交叉验证

对关键信息做 5 项验证：
1. 同一信息是否出现在 ≥2 个独立来源？
2. 官方说法与媒体报道是否矛盾？
3. 股权结构是否存在"壳公司链条"？
4. 核心人物是否频繁在多家关联公司间切换？
5. 资金流向是否存在循环交易？

### Phase 6 — 输出报告

按报告模板输出（见下节）。

## 六、实战技巧

| 场景 | 策略 |
|------|------|
| 上市公司 | 优先抓新浪财经、东方财富、同花顺、巨潮资讯 |
| 非上市公司 | 行业网站、头条搜索、微信公众号、政府公示 |
| 天眼查/企查查被拦截 | 换百度爱企查、国家企业信用信息公示系统 |
| Google 无结果 | 换头条搜索、行业垂直网站、直接抓官网 |
| 信息极少 | 标注"信息受限"，给出替代建议 |

## 七、深度挖掘（expand）

对报告中某个实体进行二次调查：
1. 以该实体为新的调查对象
2. 选择 3-5 个相关搜索角度
3. 提取新的实体和关系
4. 更新原报告中的关联网络部分
5. 检查 cross-link（该实体是否通过其他路径已出现在图谱中）
